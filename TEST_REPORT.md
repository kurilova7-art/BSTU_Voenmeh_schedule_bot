## Отчёт о тестировании бота расписания БГТУ «ВОЕНМЕХ»

## 1. Объект тестирования
Telegram‑бот для поиска расписания по группе, преподавателю, аудитории, с дополнительными функциями: часто задаваемые вопросы (FAQ) и новости вуза.

## 2. Виды тестирования
- **Юнит‑тестирование**
- **Ручное функциональное тестирование**

## 4. Ручное тестирование

### 4.1. Ошибки, найденные и исправленные на этапе разработки (основной функционал)

| № | Описание ошибки | Исправление |
|---|----------------|--------------|
| 1 | Первая пара в расписании отображалась в конце, нарушая хронологию | Добавлена функция `time_to_minutes` и сортировка `lessons = sorted(lessons, key=lambda l: time_to_minutes(l.time.split()[0]))` во всех функциях (`get_schedule_text`, `get_teacher_schedule_text`, `get_classroom_schedule_text`, `get_week_schedule_text`). |
| 2 | Ошибка `UnicodeDecodeError` при скачивании XML | Замена `resp.text()` на `resp.read()` и запись файла в бинарном режиме (`"wb"`). |
| 3 | Ошибка `MESSAGE_TOO_LONG` при показе недельного расписания для аудитории 468* | Добавлена функция `split_long_message()`, разбивающая текст на части до 4000 символов. |
| 4 | При поиске по преподавателю однофамильцы не уточнялись | Добавлена клавиатура выбора из списка и состояние `SearchStates.choosing_lecturer`. |
| 5 | Бот не запоминал выбранный день/неделю после перезапуска | Реализовано сохранение `user_states` в JSON‑файл через `storage.py` и загрузка при старте. |
| 6 | Клавиатура дней недели содержала воскресенье, которого нет в расписании | Удалён «Вс» из `WEEKDAYS_SHORT`, количество дней 6. |
| 7 | Неправильная обработка регистра при вводе группы / преподавателя | Добавлено `.upper()` для группы и `.title()` для преподавателя. |
| 8 | Отсутствовала группировка пар с одинаковым временем и дисциплиной для лекций потока | Добавлена группировка через словари (`grouped`) в `get_teacher_schedule_text`, `get_classroom_schedule_text`, `get_week_schedule_text`. |

### 4.2. Ошибки, найденные при тестировании функций FAQ и новостей

| №  | Описание ошибки | Исправление |
|----|----------------|--------------|
| 9  | Команда `/about` дублировала информацию, которая теперь находится в FAQ | Команда `/about` удалена, её содержимое перенесено в соответствующий пункт FAQ. |
| 10 | В обработчике FAQ не учитывался пункт со ссылками, отображался только текст без кнопок | Добавлена проверка на наличие ключа `"links"` в пункте FAQ; для таких пунктов формируется клавиатура с URL-кнопками и кнопкой «Назад». |
| 11 | Ошибка `ValueError` при нажатии на кнопку «Назад» в FAQ (`invalid literal for int() with base 10: 'back'`) | Фильтр для обработчика `faq_show_answer` изменён с `F.data.startswith("faq_")` на `F.data.regexp(r'^faq_\d+$')`, чтобы исключить срабатывание на `faq_back`. |
| 12 | Длинные заголовки новостей не помещались на кнопках | Увеличен лимит обрезания заголовка до 64 символов (максимально допустимая длина текста кнопки в Telegram). |

## 5. Неисправленные ошибки / ограничения

- Бесплатный сервер (Railway) может ограничивать время работы, поэтому для 24/7 рекомендуется платный тариф.
- При получении новостей через WordPress REST API возможны задержки, если сайт военмеха недоступен – бот выводит сообщение об ошибке.

## 6. Вывод

Все заявленные функции (основное расписание, FAQ, новости) работают корректно. 
Выявленные в процессе тестирования ошибки исправлены. Юнит-тесты покрывают ключевые вспомогательные функции. 
Бот готов к эксплуатации.
